# ATK Mod Pages

Source of truth for Antikythera Studios mod description pages. Author each page
**once** here; the `atk-publisher` desktop editor renders it to **Modrinth**
markdown and **CurseForge** HTML, ready to paste into each site, in one unified,
sectioned layout — the style used by projects like
[pixelshot](https://www.curseforge.com/minecraft/mc-mods/pixelshot).

> This `pages/` tree is additive and independent from the Epic Fight resources
> elsewhere in this repo.

## Layout

```
pages/
  shared/
    icons/       # social/link buttons served via raw.githubusercontent
  data/
    <mod>/
      mod.yml            # links + the ordered, typed section manifest + divider
      sections/*.md      # one file per MARKDOWN section (banner/media are images)
      assets/            # this mod's banner, media and divider images
      build/             # GENERATED by the tool — do not edit by hand
```

Copy `data/_template/` to `data/<your-mod>/` to start a new page, or use the
editor's **New** button.

## Sections

`mod.yml` holds an ordered `sections:` manifest. Each entry has a `type`:

- **markdown** — an editable `.md` file (`about`, `features`, `installation`,
  `configuration`, `social`, `credits`).
- **banner** — a single header image, always centered (recommended 1720x500).
- **media** — a list of images under a `📷 Media` heading, always centered
  (recommended 1920x1080).

An optional top-level `divider:` image (recommended 300x450, centered) is drawn
between every section. Banner, media and divider are images, not `.md` files.

Markdown sections use two placeholders:

| Placeholder            | Expands to                                            |
|------------------------|-------------------------------------------------------|
| `{{links}}`            | the icon button row (only links present in `mod.yml`) |
| `{{section: Title}}`   | a section heading for `Title`                          |

Reference images with a relative path under `assets/`, e.g.
`![shot](assets/shot.png)`. The tool rewrites these to public
`raw.githubusercontent.com` URLs. Alignment is emitted per platform
(`<center>` for Modrinth, `text-align`/`float` styles for CurseForge).

## Why two renderers

Modrinth renders GitHub-flavored markdown directly, while CurseForge's editor
takes an HTML subset. The editor produces both from this one source: a
`build/modrinth.md` and a `build/curseforge.html`, each ready to paste.
