# ATK Mod Pages

Source of truth for Antikythera Studios mod description pages. Author each page
**once** here; the private `atk-publisher` tool renders it to **Modrinth**
(markdown, published via API) and **CurseForge** (HTML, pasted into the editor)
in one unified, sectioned layout — the style used by projects like
[pixelshot](https://www.curseforge.com/minecraft/mc-mods/pixelshot).

> This `pages/` tree is additive and independent from the Epic Fight resources
> elsewhere in this repo.

## Layout

```
pages/
  shared/
    icons/       # social/link buttons (website, youtube, patreon, modrinth,
                 #   github, discord, curseforge) — served via raw.githubusercontent
    banners/     # optional decorative section-header strips
    templates/   # snippet references
  data/
    <mod>/
      mod.yml            # ids, slugs, links, section order
      sections/*.md      # page content, concatenated in filename order
      assets/            # this mod's banner + gallery images
      build/             # GENERATED by the tool — do not edit by hand
```

Copy `data/_template/` to `data/<your-mod>/` to start a new page.

## Authoring conventions

Sections are normal GitHub-flavored markdown plus two placeholders:

| Placeholder            | Expands to                                            |
|------------------------|-------------------------------------------------------|
| `{{links}}`            | the icon button row (only links present in `mod.yml`) |
| `{{section: Title}}`   | a section header for `Title`                          |

Reference images with a relative path under `assets/`, e.g.
`![banner](assets/banner.png)`. The tool rewrites these to public
`raw.githubusercontent.com` URLs automatically.

## Why two renderers

Modrinth renders GitHub-flavored markdown directly. CurseForge's editor takes an
HTML subset and has **no write API**, so its output is produced as a
paste-ready `build/curseforge.html`. One source, two targets.
